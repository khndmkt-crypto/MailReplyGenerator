# MailReplyGenerator

LM Studioのopenai/gpt-oss-20bモデルを使用して、クリップボードにコピーされた受信メールから返信メールを自動生成するツールです。

## 概要

クリップボードにコピーされた受信メールを基に、ユーザーが指定した返信内容の概要に従って、LM StudioのローカルLLMを使用して適切な返信メールを自動生成し、クリップボードにコピーします。

## 機能

- クリップボードから受信メールを自動取得
- 受信メールの言語を自動検出（英語/日本語）
- ユーザーが指定した返信概要に基づく返信メールの生成
- ビジネス適切な文体での返信生成（過剰に丁寧でない）
- 生成された返信メールをクリップボードに自動コピー
- シンプルなGUI操作

## 必要な環境

1. **LM Studio**: ローカルLLM実行環境
   - [LM Studio](https://lmstudio.ai/)をインストール
   - openai/gpt-oss-20bモデルをインストール
   - ローカルサーバーを起動（通常は http://localhost:1234）

2. **Python依存関係**:
   ```bash
   pip install -r requirements.txt
   ```

## インストール

1. このリポジトリをクローンまたはダウンロード
2. 依存関係をインストール:
   ```bash
   cd MailReplyGenerator
   pip install -r requirements.txt
   ```

## 使用方法

### 1. LM Studioの準備

1. LM Studioを起動
2. openai/gpt-oss-20bモデルをロード
3. ローカルサーバーを開始（ポート1234）

### 2. プログラムの起動

```bash
python main.py
```

### 3. 返信メールの生成

1. 返信したいメールをクリップボードにコピー
2. プログラムを起動（起動時に自動的にクリップボードから取得されます）
3. または「クリップボードから取得」ボタンをクリック
4. 「返信の概要」欄に、どのような返信をしたいかを入力
5. 「返信生成」ボタンをクリック
6. 生成された返信メールがクリップボードにコピーされ、完了通知が表示されます
7. メールクライアントに貼り付けて使用してください

### 4. 操作ボタン

- **クリップボードから取得**: クリップボードからメールを再取得
- **返信生成**: 返信メールを生成してクリップボードにコピー
- **クリア**: 表示されているメールと入力内容をクリア

## 設定

設定は `config.json` ファイルで管理されます。初回起動時に自動的に作成されます。

設定ファイルのテンプレートは `config.json.example` を参照してください。

主な設定項目：
- **lm_studio**: LM Studioの接続設定
  - `url`: APIエンドポイント（デフォルト: `http://localhost:1234/v1`）
  - `model`: 使用するモデル名（デフォルト: `openai/gpt-oss-20b`）
  - `api_key`: APIキー（デフォルト: `lm-studio`）
  - `timeout`: タイムアウト時間（秒）
- **ui**: ユーザーインターフェース設定
  - `window_width`: ウィンドウ幅（ピクセル）
  - `window_height`: ウィンドウ高さ（ピクセル）
  - `font_size`: フォントサイズ
- **generation**: 返信生成パラメータ
  - `temperature`: 温度パラメータ（0.0-1.0）
  - `max_tokens`: 最大トークン数
- **language_detection**: 言語検出設定
  - `default_language`: デフォルト言語（"auto"で自動検出）
  - `fallback_language`: 検出失敗時のフォールバック言語（"ja" または "en"）

## トラブルシューティング

### LM Studioに接続できない場合

1. LM Studioが起動していることを確認
2. ローカルサーバーがポート1234で動作していることを確認
3. openai/gpt-oss-20bモデルがロードされていることを確認

### クリップボードからテキストが取得できない場合

1. クリップボードにテキストがコピーされていることを確認
2. プログラムにクリップボードへのアクセス権限があることを確認（macOSの場合）

### 返信メールが生成されない場合

1. LM Studioが起動していることを確認
2. モデルがロードされていることを確認
3. 返信の概要が入力されていることを確認
4. ログファイル（`mail_reply_generator.log`）を確認してエラー内容を確認

### 生成された返信に不要な説明文が含まれる場合

プログラムは不要な説明文を自動的に除去しようとしますが、モデルの出力によっては完全に除去できない場合があります。その場合は、生成された返信を手動で編集してください。
