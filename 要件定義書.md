# メール返信作成プログラム 要件定義書

## 1. プロジェクト概要

### 1.1 プロジェクト名
MailReplyGenerator - LM Studioを使用したメール返信自動生成ツール

### 1.2 目的
クリップボードにコピーされた受信メールを基に、ユーザーが指定した返信内容の概要に従って、LM Studioのopenai/gpt-oss-20bモデルを使用して適切な返信メールを自動生成し、クリップボードにコピーするプログラムを開発する。

### 1.3 背景
- メール返信の作成に時間がかかる
- 適切なビジネス文体で返信メールを作成したい
- LM StudioのローカルLLMを活用してメール返信を効率化したい
- 受信メールの言語に応じて適切な言語で返信したい

### 1.4 対象ユーザー
- 日常的にメール返信を行うビジネスパーソン
- 多言語でのメール対応が必要なユーザー
- LM Studioを使用しているMacユーザー

## 2. 機能要件

### 2.1 基本機能

#### 2.1.1 クリップボード取得機能
- Macのクリップボードからテキスト（受信メール）を取得
- クリップボードが空の場合のエラーハンドリング
- テキスト以外のデータがクリップボードにある場合の処理

#### 2.1.2 ユーザー入力機能
- 返信メールの概要をユーザーに入力してもらう
- シンプルなテキスト入力インターフェース
- 入力内容の検証（空文字列チェックなど）

#### 2.1.3 メール返信生成機能
- LM Studioのopenai/gpt-oss-20bモデルを使用
- 受信メールの言語を自動検出（英語/日本語）
- 検出した言語に合わせて返信メールを作成
- ビジネスで通用するが、過剰に丁寧にならない文体
- メール返信以外の不要なtalkを排除（件名、ヘッダー、LLMからの説明文など）

#### 2.1.4 クリップボード出力機能
- 生成された返信メールをクリップボードにコピー
- 作業完了の通知（GUI通知）

### 2.2 詳細機能

#### 2.2.1 クリップボード取得
- `pyperclip`を使用してクリップボードからテキストを取得
- クリップボードが空の場合のエラーメッセージ表示
- 取得したテキストの前処理（改行の正規化など）

#### 2.2.2 言語検出
- 受信メールのテキストを分析して言語を判定
- 英語と日本語の基本的な検出
- 検出結果に基づいて返信言語を決定

#### 2.2.3 LM Studio連携
- LM Studioのローカルサーバー（デフォルト: `http://localhost:1234/v1`）に接続
- openai/gpt-oss-20bモデルを指定してAPI呼び出し
- OpenAI互換のAPI形式を使用
- プロンプトの構築：
  - 受信メールの内容
  - ユーザーが指定した返信概要
  - 言語指示（英語/日本語）
  - 文体指示（ビジネス適切、過剰に丁寧でない）
  - 不要なtalkを排除する指示（件名、ヘッダー、LLMからの説明文など）
- エラーハンドリング（接続失敗、タイムアウトなど）

#### 2.2.4 ユーザーインターフェース
- **GUIフレームワーク**: tkinter（標準ライブラリ）
- **レイアウト**:
  - 上部: 受信メール表示エリア（読み取り専用）
  - 中部: 返信概要入力エリア（テキスト入力）
  - 下部: 操作ボタン（生成実行、クリアなど）
  - ステータス表示エリア: 処理状況や完了通知を表示
- **操作**:
  - 「返信生成」ボタン: 返信メールを生成してクリップボードにコピー
  - 「クリア」ボタン: 入力内容をクリア
  - 「クリップボードから取得」ボタン: クリップボードからメールを再取得
- **通知**:
  - 作業完了時: GUI上に完了メッセージを表示
  - エラー発生時: GUI上にエラーメッセージを表示
  - 処理中: ステータス表示エリアに処理状況を表示

### 2.3 エラーハンドリング
- LM Studioに接続できない場合のエラーメッセージ表示
- クリップボードが空の場合の警告
- 返信生成API呼び出し失敗時のエラー表示
- ネットワークタイムアウトの処理
- モデルがロードされていない場合の警告
- 言語検出失敗時のフォールバック処理

## 4. 技術要件

### 4.1 開発環境
- **プログラミング言語**: Python 3.8以上
- **OS対応**: macOS
- **GUIフレームワーク**: tkinter（標準ライブラリ）

### 4.2 必要なライブラリ
- **クリップボード操作**: `pyperclip`
- **HTTP通信**: `requests`
- **GUI**: `tkinter`（標準ライブラリ）
- **ログ**: `logging`（標準ライブラリ）
- **設定管理**: `json`（標準ライブラリ）、`pathlib`（標準ライブラリ）
- **言語検出**: `langdetect`（オプション、シンプルな実装でも可）

### 4.3 アーキテクチャ
- **シンプルな関数型設計**
- **モジュール構成**:
  - `clipboard_manager.py`: クリップボード操作
  - `mail_processor.py`: メール処理（言語検出など）
  - `reply_generator.py`: LM Studio API連携と返信生成
  - `gui.py`: ユーザーインターフェース
  - `config.py`: 設定管理
  - `main.py`: メインエントリーポイント

### 4.4 LM Studio設定
- **APIエンドポイント**: `http://localhost:1234/v1`
- **モデル名**: openai/gpt-oss-20b（実際のモデル名は設定ファイルで指定可能）
- **API形式**: OpenAI互換（`/chat/completions`）
- **認証**: Bearer token（デフォルト: `lm-studio`）

## 6. プロンプト設計

### 6.1 プロンプトテンプレート
返信生成時のプロンプトは以下の構造とする：

```
あなたはビジネスメールの返信作成を支援するアシスタントです。
以下の受信メールに対して、ユーザーが指定した内容に基づいて返信メールを作成してください。

【受信メール】
{受信メールの内容}

【返信の概要】
{ユーザーが入力した返信概要}

【指示】
- 返信メールの本文のみを出力してください。それ以外の一切のテキストは不要です。
- 以下のような不要な内容は一切出力しないでください：
  * 件名やメールヘッダー
  * 「わかりました、以下のような返信を作成しました」といった説明文
  * 「以下が返信メールです」といった前置き
  * その他、返信メール本文以外のすべてのテキスト
- 受信メールが英語の場合は英語で、日本語の場合は日本語で返信してください。
- ビジネスで通用する適切な文体で、過剰に丁寧にならない程度の丁寧さで作成してください。
- 出力は返信メールの本文のみで、それ以外の文字は一切含めないでください。
```

## 7. 使用方法

### 7.1 起動方法
```bash
python main.py
```

### 7.2 操作手順
1. LM Studioを起動し、openai/gpt-oss-20bモデルをロード
2. ローカルサーバーを開始（ポート1234）
3. 返信したいメールをクリップボードにコピー
4. 本プログラムを起動
5. クリップボードからメールが自動取得される（または「クリップボードから取得」ボタンをクリック）
6. 返信の概要を入力エリアに入力
7. 「返信生成」ボタンをクリック
8. 生成された返信メールがクリップボードにコピーされ、完了通知が表示される

## 8. 制約事項

### 8.1 技術的制約
- LM Studioが起動している必要がある
- openai/gpt-oss-20bモデルがロードされている必要がある
- ローカルサーバーがポート1234で動作している必要がある
- インターネット接続は不要（ローカルLLMを使用）

### 8.2 機能制約
- 処理対象はテキストのみ（画像や添付ファイルは非対応）
- 長文のメールの場合は処理時間がかかる可能性がある
- 返信品質はopenai/gpt-oss-20bモデルの性能に依存
- 言語検出は英語と日本語を主に対象とする
